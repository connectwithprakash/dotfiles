# User-Level CLAUDE.md

Global instructions for Claude Code across all projects.

## Commit and Documentation Standards

- Write commit messages focusing on WHAT changed and WHY, not HOW
- Document architectural decisions and trade-offs in code comments
- Maintain clear separation between experimental and production code

## Code Style

- Start simple, add complexity only when demonstrably required
- Prefer deterministic logic for control flow
- Test-driven development for production code
- Implement human-in-the-loop checkpoints for critical decisions
